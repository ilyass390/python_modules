Q: What is a virtual environment and why is it important?

A venv is an isolated Python environment with its own packages and Python binary. It prevents conflicts between projects that need different versions of the same library.

Q: What problems do virtual environments solve?

Dependency conflicts — e.g. Project A needs pandas 1.0 and Project B needs pandas 2.0. Without venvs they'd clash in the global environment.

Q: How does your program detect the virtual environment?

By comparing sys.prefix and sys.base_prefix. When a venv is active, sys.prefix points to the venv folder.
sys.base_prefix always points to the real system Python. If they differ → we're in a venv.
